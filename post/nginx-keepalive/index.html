# coer_module
# 单个conn最多可以服务的请求数；(超出将关闭连接)
# 大值造成memory usage;
# 这个参数的真实含义，是指一个keep alive建立之后，nginx就会为这个连接设置一个计数器，记录这个keep alive的长连接上已经接收并处理的客户端请求的数量。如果达到这个参数设置的最大值时，则nginx会强行关闭这个长连接，逼迫客户端不得不重新建立新的长连接。
Syntax:	  keepalive_requests number;
Default:	keepalive_requests 100;
Context:	http, server, location

# 空闲超时时间
Syntax:	  keepalive_timeout timeout [header_timeout];
Default:	keepalive_timeout 65s;
Context:	http, server, location

nginx: keepalive_timeout 含义

  • nginx端连接建立后的超时时间、并没有keepalive check探测；到时间后nginx端发送[FIN];

  • 四次挥手最后发包变为 RST，half-close;

  • 如下图：浏览器45s发送keep-alive 探测包；

  • 服务端主动断开：RST

  • 客户端复用连接：

  • 浏览器访问不存在port – RST (Connection refused)

	server   192.168.0.1：8080  weight=1 max_fails=2 fail_timeout=30s;
	server   192.168.0.2：8080  weight=1 max_fails=2 fail_timeout=30s;

}

server {
    location /http/ {
        proxy_pass http://http_backend;

        proxy_set_header Connection "";
    }
}

# Sets the maximum number of requests that can be served through one keepalive connection. After the maximum number of requests is made, the connection is closed.
Syntax:	keepalive_requests number;
Default:	keepalive_requests 100;
Context:	upstream

# ---
# Sets a timeout during which an idle keepalive connection to an upstream server will stay open.
Syntax:	keepalive_timeout timeout;
Default:	keepalive_timeout 60s;
Context:	upstream

  • 闲置连接建立反复震荡：

    • 理想情况：100ms响应时间、1wQPS、并发1k连接，当请求过来平稳保持1k，无idle连接；
    • 当请求过来不平稳，需要反复删除多余idle连接(前10ms多，后10ms少)、重新建立新连接(前10ms少、后10ms多)；
    • 一般设置为最大需要长连接 * （10% - 30%）

  • nginx端 TIME_WAIT过多：

    • TIME_WAIT出现在主动关闭端；
    • keepalive_request: 设置过小，当超出该值，连接主动关闭；
    • Keepalive xxx: 设置过小，高并发下反复震荡，不断关闭、重建连接；
