VXLAN的特点是将L2的以太帧封装到UDP报文（即L2 over L4）中，并在L3网络中传输。

[toc]

VXlan

  • 面临挑战：

    • 虚拟机网络规模受限：

      大二层网络通过arp寻址转发，受表大小限制；

    • 网络隔离能力限制：

      • Vlan 12位，4096个，无法满足多租户规模；

    • 虚拟机迁移受网络架构限制：

      • 虚拟机迁移需要保证mac、ip地址不变；vxlan相当于一层抽象层，保证VTEP设备IP、Mac地址不变；

  • 解决的问题：

    • vxlan将数据包封装为udp包，再使用物理网络ip、mac作为外层封装，对网络只表现为封装后参数；
    • 隔离能力增加 2^24
    • 保证虚拟机迁移ip、mac不变；

  1. vtep收到请求

    1. 多播组中所有主机都收到报文，内核发现是vxlan报文，根据vni发送给对应vtep;

    3. 将收到的mac, vetp主机IP写入fdb表；

    2. vtep解包将arp缓存到内核，vtep主机写入fdb;

  3. ping请求交互

    id 42 \
    dstport 4789 \
    group 239.1.1.1 \
    local 192.168.8.100 \
    dev eth1

# bridge设备，vetp绑定
ip link add br0 type bridge
ip link set vxlan0 master bridge
ip link set vxlan0 up
ip link set br0 up


# 创建独立network ns;
ip netns add container1

# 创建 veth pair，并把一端加到网桥上
ip link add veth0 type veth peer name veth1
ip link set dev veth0 master br0
ip link set dev veth0 up


# 配置容器内部的网络和 IP
ip link set dev veth1 netns container1
ip netns exec container1 ip link set lo up

ip netns exec container1 ip link set veth1 name eth0
ip netns exec container1 ip addr add 10.20.1.2/24 dev eth0
ip netns exec container1 ip link set eth0 up
